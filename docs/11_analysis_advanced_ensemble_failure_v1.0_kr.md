# 유저별 차등 앙상블 실패(0.1100) 및 향후 전략 분석

**문서 버전:** v1.0  
**작성일:** 2026-02-08  
**작성자:** Antigravity  
**관련 태스크:** Ensemble Score Optimization

---

## 1. 현상 분석

| 모델 | 가중치 설정 | 결과 (NDCG@10) |
|---|---|---|
| **Simple Ensemble** | 全 유저: **ALS 0.7 + SASRec 0.3** | **0.1150 (Best)** |
| **Advanced Ensemble** | Short(<5): **ALS 1.0** / Long(>=5): **ALS 0.6 + SASRec 0.4** | 0.1100 (▼ -0.005) |

### 1.1 왜 점수가 떨어졌는가?

우리는 "인터랙션 5개 미만의 Short Sequence 유저에게 SASRec은 노이즈일 것이다"라고 가정하고 **가중치를 0으로 만들었지만, 이것이 패착**이었습니다.

**이유 1: Short Sequence에서도 "마지막 아이템" 정보는 강력했다.**
-   ALS는 전체적인 행렬 분해로 "이 유저가 좋아할 만한 아이템"을 찾지만, **"방금 본 아이템 다음에 뭘 볼지"**에 대한 정보는 약합니다.
-   SASRec은 `[A, B]`라는 짧은 시퀀스에서도 **"B 다음에 주로 보는 C"**라는 강력한 **컨텍스트 신호**를 보냈을 것입니다.
-   우리가 이를 제거(ALS 100%)함으로써, Short User들에 대한 예측력이 오히려 떨어졌습니다.

**이유 2: Long User에게 0.4 가중치는 과했다.**
-   SASRec 자체 성능(0.08)이 ALS(0.11)보다 확실히 낮습니다.
-   Long User라 하더라도 SASRec의 비중을 0.3에서 0.4로 높인 것이 **ALS의 안정적인 예측력을 희석**시켰을 가능성이 큽니다.

## 2. 결론 및 향후 전략

**결론:**  
SASRec은 비록 전체 리더보드 점수는 낮지만, **모든 유저 세그먼트에서 ALS가 놓치는 "순차적(Contextual) 정보"를 보완해주고 있음**이 입증되었습니다. 따라서 특정 유저군에서 이를 아예 배제하는 것은 좋지 않습니다.

### 🚀 점수 상승을 위한 제언

이제 "복잡한 세그먼트 전략"보다는 **"전역 가중치(Global Weight) 미세 조정"**으로 돌아가야 합니다. 
현재 **0.1150 (0.7:0.3)**이 매우 훌륭한 균형점(Local Optimum)일 수 있습니다.

**다음 두 가지 실험을 제안합니다:**

1.  **가중치 미세 조정 (Weight Tuning)**
    -   `ALS 0.8 : SASRec 0.2` (SASRec 비중 축소)
    -   `ALS 0.6 : SASRec 0.4` (SASRec 비중 확대 - 전체 유저 대상)
    -   이 두 가지를 제출해보고 0.1150보다 오르는 쪽으로 방향을 잡습니다.

2.  **보수적 세그먼트 전략 (Conservative Segmentation)**
    -   Short User에게 아예 0을 주는 대신, **가중치를 약간만 줄이는** 방식입니다.
    -   예: Short(0.8:0.2), Long(0.7:0.3)

**가장 추천하는 방법:**
-   단순하게 **0.1150 (ALS 0.7 + SASRec 0.3)** 결과를 **최종 베스트**로 확정하고 마무리하는 것도 좋은 전략입니다. 
-   만약 더 욕심을 내고 싶다면, `ensemble.py`의 `w_als` 파라미터만 0.8 / 0.6으로 바꾸어 두 번 더 제출해보는 것이 가장 효율적입니다.
