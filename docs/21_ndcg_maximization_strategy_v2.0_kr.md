# nDCG@10 점수 극대화 전략 보고서 v2.0

## 현재 상태
- **현재 최고점**: 0.1344 (4-Way Ensemble + Phase 1+ Enhanced Boost)
- **모델 구성**: ALS(0.5) + SASRec(0.25) + XGBoost(0.10) + CatBoost(0.15)
- **후처리**: Repeat Boost + Event Type + Recency + Top-3 Boost

---

## nDCG@10 직관적 이해

nDCG@10 = $\frac{DCG@10}{IDCG@10}$, 여기서 $DCG@10 = \sum_{i=1}^{10}\frac{rel_i}{\log_2(i+1)}$

**핵심 직관:**
1. **위치 가중치**: 1위 아이템은 2위보다 ~63% 더 큰 가중치 ($\frac{1}{\log_2(2)}$ vs $\frac{1}{\log_2(3)}$)
2. **정확도 > 포괄성**: 10개 중 1개라도 맞추면 점수가 나오지만, 그 1개가 **1위에 있으면** 최대 효과
3. **순서가 핵심**: 동일한 정답 아이템 집합이라도 상위에 배치할수록 점수 급등

**따라서 전략 방향:**
- ❌ "더 많은 정답을 찾자" (recall 증가만으로 한계)
- ✅ **"확실한 정답을 1~3위에 올리자"** (precision at top positions)

- **카테고리/브랜드/가격** 완전 미활용 → **가장 큰 기회**

## 6대 전략

### 전략 1: 카테고리 친화도 부스트 (예상 영향: ⭐⭐⭐⭐⭐)

**근거**: 유저당 카테고리 중앙값이 단 2개. 유저의 관심 카테고리는 매우 집중됨.

**방법**:
```
유저u의 상위 카테고리 = category_code별 상호작용 비율 계산
후보 아이템i의 카테고리가 유저 top-1 카테고리 → boost × 1.5
후보 아이템i의 카테고리가 유저 top-2 카테고리 → boost × 1.25
```

**직관**: "최근에 전자제품만 보던 유저에게 전자제품을 더 높이 추천"
→ nDCG 관점: 정답 아이템이 유저의 주 카테고리일 확률이 높으므로, 이를 상위로 끌어올림

---

### 전략 2: 브랜드 충성도 부스트 (예상 영향: ⭐⭐⭐⭐)

**근거**: 1,859개 브랜드 중 유저가 반복 조회하는 브랜드는 구매 확률이 높음

**방법**:
```
유저u의 브랜드별 조회 횟수 계산
top-1 브랜드 아이템 → boost × 1.3
top-2~3 브랜드 아이템 → boost × 1.15
```

**직관**: "삼성 제품만 보던 사람은 삼성 신제품을 살 확률이 높다"

---

### 전략 3: 유저 세그먼트별 차등 전략 (예상 영향: ⭐⭐⭐⭐)

**근거**: 전체 유저에게 동일한 부스트를 적용하는 것은 비효율적

| 세그먼트 | 비율 | 전략 |
|----------|------|------|
| **Light (1-5건)** | 47.4% | 개인화 효과 없음 → **인기도/트렌딩 부스트 강화**, 개인 부스트 약화 |
| **Medium (6-20건)** | 35.4% | 표준 Phase 1+ 부스트 유지 |
| **Heavy (21+건)** | 17.2% | 개인 히스토리 풍부 → **카테고리/브랜드 부스트 강화**, 인기도 약화 |

**직관**: 처음 온 손님에게는 베스트셀러를 추천하고, 단골에게는 취향 맞춤 추천

---

### 전략 4: 트렌딩 아이템 부스트 (예상 영향: ⭐⭐⭐)

**근거**: 최근 14일 Top-200 중 40%가 새로 부상한 아이템. 인기 아이템은 빠르게 변함.

**방법**:
```
최근 14일 인기도 순위 vs 이전 인기도 순위 비교
순위 상승 아이템 → trending_boost = 1 + 0.3 × (순위 상승 비율)
기존 모델에 반영 안 된 신규 인기 아이템을 상위로 끌어올림
```

**직관**: "요즘 뜨는 제품"은 갑자기 인기가 올라간 것 → 기존 모델이 반영 못함

---

### 전략 5: 가격대 친화도 부스트 (예상 영향: ⭐⭐⭐)

**근거**: 가격 정보 완전 미활용. 유저마다 선호 가격대가 존재.

**방법**:
```
유저u의 평균 조회 가격 = avg_price
후보 아이템 가격이 avg_price × 0.5 ~ avg_price × 2.0 범위 → boost × 1.15
범위 밖 → boost × 0.9 (페널티)
```

**직관**: "보통 5만원대 제품을 보는 유저에게 200만원짜리 추천은 비효율적"

---

### 전략 6: 세션 기반 공동조회 부스트 (예상 영향: ⭐⭐⭐)

**근거**: 같은 세션에서 함께 조회된 아이템은 관련성이 높음 (비교 쇼핑, 연관 상품)

**방법**:
```
아이템 A,B가 같은 세션에서 함께 조회된 횟수 = coview(A,B)
유저의 최근 조회 아이템과 co-view 점수가 높은 후보 → boost × 1.2
```

**직관**: "같은 쇼핑 세션에서 비교한 제품들은 서로 대체재"

## 리스크 분석

1. **과적합 위험**: 부스트가 너무 많으면 오히려 다양성 감소 → 부스트 값을 보수적(1.1~1.3)으로 시작
2. **카테고리 리크 위험**: 테스트 데이터의 정답이 다른 카테고리일 수 있음 → 페널티 없이 부스트만 적용
3. **Light 유저 처리**: 47%가 히스토리 부족 → 인기도 기반 폴백 필수
