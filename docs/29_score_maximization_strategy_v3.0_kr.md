# nDCG@10 점수 극대화 전략 v3.0

## 현재 상태 (2025.01)

| 지표 | 값 |
|------|-----|
| **현재 최고 nDCG@10** | **0.1344** |
| 최고 모델 구성 | 4-Way Ensemble + Phase 1+ Enhanced |
| 베이스라인 대비 | +14.6% (0.1173 → 0.1344) |
| 사용 데이터 컬럼 | `user_id`, `item_id`, `event_type`, `event_time` (4개만) |
| **미사용 컬럼** | **`category_code`, `brand`, `price`, `user_session`** |

---

## 핵심 진단: 왜 0.1344에서 정체되는가?

### 1. 모델은 이미 강하다
- ALS(0.1124) + SASRec(0.0864) + XGB(0.1117) + CatBoost(0.1125) → 앙상블=0.1341
- 4개 모델이 각각 다른 관점으로 추천 (다양성이 핵심)
- Post-processing(Phase 1+)은 단 +0.22%만 추가

### 2. 학습 과정에서 이미 캡처된 시그널
- ALS: user-item 잠재벡터가 카테고리/브랜드를 **암묵적으로** 인코딩
- CatBoost: `item_cat`, `item_brand`를 **명시적 피처**로 사용 중
- SASRec: 시퀀셜 패턴으로 카테고리 전이를 **부분 학습**

### 3. 남은 기회: 모델이 캡처하지 **못하는** 시그널
| 시그널 | 왜 모델이 놓치나 | 활용 방안 |
|--------|-----------------|----------|
| 세션 내 문맥 | ALS는 세션 개념 없음, SASRec는 전체 시퀀스 | 마지막 세션의 카테고리 집중 |
| 유저 세그먼트 차등 | 모든 유저를 동일하게 처리 | Light/Heavy 차등 전략 |
| 가격 범위 이탈 | 순위 모델이라 가격 필터 없음 | 극단적 가격 이탈 패널티 |
| 전환률 기반 우선순위 | view→purchase 전환 확률 미반영 | 고전환 아이템 우대 |

## 3대 전략 프레임워크

### 전략 1: 개인화된 추천 강화 (Personalization)

> "과거 구매 이력, 검색/클릭 행동 기반 맞춤 추천"

**현재 반영 수준:** ★★★☆☆
- ✅ Repeat Boost (과거 상호작용 아이템 우대)
- ✅ Event Type 가중치 (purchase > cart > view)
- ❌ 카테고리 선호도 미반영 (68.4% 일관성 미활용!)
- ❌ 브랜드 충성도 미반영
- ❌ 가격 친화도 미반영

**제안 액션:**
```
A1. 카테고리 친화도 부스트 (Conservative)
    - 유저의 Top-1 카테고리 매칭 아이템: ×1.15
    - 유저의 Top-2 카테고리 매칭 아이템: ×1.08
    → 근거: 68.4% 카테고리 일관성, but 모델이 50%+ 이미 캡처
    → 핵심: 1.15배로 '보조'만 함 (1.4배는 과도 → Phase 2 실패 교훈)

A2. 브랜드 충성도 부스트
    - 유저의 Top-1 브랜드 매칭: ×1.10
    - 유저의 Top-2/3 브랜드 매칭: ×1.05
    → 브랜드는 카테고리보다 약한 시그널

A3. 가격 범위 가드
    - 유저 평균가격 ± 3σ 벗어나면: ×0.85 (패널티)
    → 극단적 가격 이탈만 걸러냄 (gentle guard)
```

### 전략 2: 피처 엔지니어링 집중 (Feature Engineering)

> "유저/아이템의 다양한 특성 활용"

**현재 반영 수준:** ★★☆☆☆
- ✅ 상호작용 횟수 (count)
- ✅ 이벤트 타입, 시간
- ❌ 아이템 전환률 (view→purchase 비율)
- ❌ 유저 세그먼트별 차등 전략
- ❌ 세션 기반 공동구매 패턴

**제안 액션:**
```
B1. 아이템 전환률 부스트
    - 각 아이템의 purchase_count / view_count = conversion_rate
    - 유저가 view만 한 아이템 중 높은 전환률 → ×1.12
    → "이 아이템을 본 사람의 30%가 구매했다" = 강력한 신호

B2. 유저 세그먼트 차등
    - Light (≤5 interactions, 42%):
      · 개인 부스트 약화 (데이터 부족 → 노이즈)
      · 인기/트렌딩 아이템 부스트 ×1.15
    - Heavy (≥20 interactions):
      · 개인 부스트 강화 (카테고리/브랜드 신뢰도 높음)
    - Medium: 기본 유지

B3. 세션 내 공동열람 부스트
    - 같은 세션에서 함께 본 아이템 쌍 분석
    - 유저가 A를 구매하면, A와 자주 함께 본 B를 부스트
    → 주의: 메모리 집약적, 제한적 적용
```

### 전략 3: 시간적 요소 고려 (Temporal Factors)

> "시간에 따른 선호도 변화, 계절성, 최근성"

**현재 반영 수준:** ★★★☆☆
- ✅ Recency 3단계 (1h/24h/72h)
- ✅ ALS decay_rate=0.1
- ❌ 마지막 세션 문맥 (48초 median 버스트)
- ❌ 트렌딩 아이템 감지

**제안 액션:**
```
C1. 마지막 세션 카테고리 강조
    - 유저의 마지막 세션(≤5분 간격) 카테고리 식별
    - 해당 카테고리 아이템: ×1.20
    → 세션 폭발성(48초 median) 활용
    → "방금 스마트폰 봤으면 스마트폰 케이스 추천"

C2. 트렌딩 아이템 부스트
    - 최근 14일 인기 급상승 아이템 (일평균 비교)
    - Light 유저에게만 약하게 부스트: ×1.10
    → Heavy 유저는 개인 취향이 더 중요

C3. 시간대 가중치 (실험적)
    - 유저의 주 활동 시간대에 인기 아이템 구분
    → 효과 불확실, 낮은 우선순위
```

## 실행 계획: 5단계 체계적 테스트

### Phase A: Ultra-Conservative (위험 최소)
**Phase 1+ 유지 + 카테고리만 약하게 추가**
```
cat_top1=1.10, cat_top2=1.05
→ 기대: 0.1344 ~ 0.1350 (현재 유지 or 미세 개선)
```

### Phase B: Conservative Personalization
**카테고리 + 브랜드 + 가격가드**
```
cat_top1=1.15, cat_top2=1.08
brand_top1=1.10, brand_top2=1.05
price_guard=0.85 (±3σ 이탈시)
→ 기대: 0.1345 ~ 0.1355
```

### Phase C: Segment-Differentiated
**Phase B + 유저 세그먼트 차등**
```
Light유저: 개인부스트 0.7배 감소, 트렌딩 1.15
Heavy유저: 개인부스트 1.3배 강화
→ 기대: 0.1350 ~ 0.1365
```

### Phase D: Session Context
**Phase C + 마지막 세션 카테고리**
```
last_session_category_boost=1.20
→ 기대: 0.1355 ~ 0.1375
```

### Phase E: Full Kitchen Sink
**모든 전략 통합 + 전환률 부스트**
```
A1+A2+A3 + B1+B2 + C1+C2 전부
→ 기대: 0.1350 ~ 0.1380
⚠️ 과적합 리스크 있음
```

## 위험 분석

| 위험 | 확률 | 대응 |
|------|------|------|
| 카테고리 부스트가 모델과 중복 | 높음 | Ultra-conservative(1.10x)부터 시작 |
| 너무 많은 부스트가 상호 간섭 | 중간 | Phase A→E 순차 테스트 |
| Light 유저 세그먼트 차등이 노이즈 | 중간 | 효과 없으면 즉시 제거 |
| 세션 정의가 데이터와 불일치 | 낮음 | 5분 간격 기준, 유연하게 조정 |

## 핵심 원칙 (실패 경험에서 배운 것)

1. **모델 다양성 > 부스트 강도**: 4-Way가 +0.83%, Phase 1+이 +0.22%
2. **보수적 부스트가 안전**: Phase 3(강한 부스트)는 -0.6% 하락
3. **한 번에 하나만**: 복합 변경은 원인 파악 불가
4. **미사용 데이터가 금맥**: 4개 컬럼 미사용 → 잠재 개선 여지
5. **42% Light 유저**: 개인 부스트가 이 유저들에겐 노이즈 → 차등 필수
