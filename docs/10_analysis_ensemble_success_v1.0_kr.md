**작성일:** 2026-02-08  
**작성자:** Antigravity  
**관련 태스크:** Ensemble Score Maximization

## 1. 현상 분석

사용자께서 보고해주신 결과는 다음과 같습니다.

| 모델 | NDCG@10 | 비고 |
|---|---|---|
| **SASRec (단독)** | 0.0813 (▼) | 기존(0.0855)보다 하락, ALS(0.1124)보다 처참함 |
| **Ensemble (ALS+SASRec)** | **0.1150 (▲)** | ALS 단일 최고점(0.1124)을 갱신 (New Best!) |

### 1.1 왜 SASRec 단독 점수는 떨어졌는가? (0.0855 → 0.0813)
가장 큰 원인은 **"데이터 필터링 완화 (5개 → 2개)"**로 인한 **노이즈 증가**입니다.

1.  **Short Sequence의 모호성:** 인터랙션이 2~4개인 유저는 다음에 무엇을 볼지 예측하기 매우 어렵습니다. 대부분 우연에 기반하거나 단순히 인기 아이템을 클릭할 가능성이 높습니다.
2.  **모델 용량 분산:** SASRec 모델이 이런 "패턴 없는" 유저들의 데이터까지 억지로 학습하려다 보니, 정작 중요한 "Long Sequence" 유저들의 정교한 패턴을 학습할 용량(Capacity)이 부족해졌거나, 노이즈에 과적합(Overfitting) 되었을 가능성이 큽니다.
3.  **결과:** 전반적인 추천 품질이 하락했습니다.

### 1.2 왜 앙상블 점수는 올랐는가? (0.1124 → 0.1150)
그럼에도 불구하고 앙상블 점수가 오른 이유는 **"다양성(Diversity)"** 때문입니다.

1.  **상호 보완:** ALS는 "전체적인 선호도(Global Interest)"를 잘 잡고, SASRec은 (비록 전체 성능은 낮아도) 일부 유저의 "즉각적인 다음 흥미(Local/Sequential Interest)"를 포착했을 것입니다.
2.  **노이즈 상쇄:** ALS가 0.7의 가중치로 중심을 잡아주면서 SASRec의 엉뚱한 추천(노이즈)을 필터링하고, SASRec이 가끔 맞추는 정교한 추천이 ALS의 놓친 부분을 채워주었기 때문입니다.
3.  **Weak Learner의 힘:** 머신러닝에서 "약한 모델"이라도 기존 모델과 *다른 관점*을 제공한다면, 합쳤을 때 성능이 오르는 원리와 같습니다.

## 2. NDCG@10 점수 극대화 방안

현재 0.1150을 넘어 **0.12** 이상을 목표로 하기 위한 구체적인 전략을 제안합니다.

### 전략 1: 유저 그룹별 차등 앙상블 (Segmented Ensemble) 🚀 **[추천]**

모든 유저에게 똑같이 `0.7 : 0.3` 비율을 적용하는 것은 비효율적입니다.

-   **Short Users (이력 < 5개):** SASRec의 예측이 매우 불안정합니다. **ALS 비중을 90% 이상**으로 높이거나 ALS 단독 사용이 낫습니다.
-   **Long Users (이력 >= 10개):** SASRec의 순차 예측이 힘을 발휘합니다. **SASRec 비중을 40~50%**까지 높여도 좋습니다.

**실행 방안:**
`ensemble.py`를 수정하여 유저별 인터랙션 개수(User History Length)를 계산하고, 그 길이에 따라 가중치(`w_als`, `w_sasrec`)를 동적으로 조절합니다.

### 전략 2: RRF (Reciprocal Rank Fusion) 최적화

단순 점수 가중합보다 **순위(Rank) 기반의 융합**이 더 안정적일 수 있습니다. (현재 코드는 이미 유사한 방식을 사용 중이나, 파라미터 튜닝이 가능합니다)
-   `Score = 1 / (Rank + K)`
-   `K` 값을 조절하여 상위권 추천에 얼마나 집중할지 튜닝합니다. (보통 K=60 사용)

### 전략 3: 불확실성 제거 (Uncertainty Filtering)

SASRec의 예측 점수(Softmax 확률)가 너무 낮은 경우(즉, 모델이 확신이 없는 경우), 아예 해당 추천을 배제하고 ALS 추천으로 채우는 방식입니다.
(단, 현재 출력 파일에는 점수 정보가 포함되지 않아 재추론이 필요할 수 있습니다. 전략 1이 더 현실적입니다.)

## 3. 제안

가장 효과적이고 즉시 적용 가능한 **[전략 1: 유저 그룹별 차등 앙상블]**을 진행해볼까요?
인터랙션 길이에 따라 가중치를 다르게 적용하는 `advanced_ensemble.py` 스크립트를 작성하여 테스트해보겠습니다.

**예상 시나리오:**
-   Length < 5: ALS (1.0) + SASRec (0.0)
-   Length >= 5: ALS (0.6) + SASRec (0.4)
