## 🔍 SASRec 성능 저하 원인

### 1. 시퀀스 길이 부족 (Critical)

SASRec은 Self-Attention 기반으로 **시퀀스 패턴**을 학습합니다.

```
이상적인 시퀀스: item1 → item2 → item3 → ... → item20 → [예측]
현실 데이터:     item1 → item2 → item3 → [예측] (너무 짧음!)
```

- **47.5%의 사용자가 5개 이하 인터랙션**
- 중앙값 6개 = Self-Attention이 의미 있는 패턴을 학습하기 어려움
- MAX_ITEM_LIST_LENGTH=50 설정했지만, 대부분 그보다 훨씬 짧음

### 2. 구매 데이터 극심한 희소성

```
구매 이벤트: 2,076건 / 8,350,311건 = 0.02%
```

- 테스트 데이터는 **구매 예측**을 평가
- 학습 데이터에서 **구매 패턴**을 찾기에 샘플 부족
- SASRec이 "구매로 이어지는 시퀀스"를 학습하기 어려움

### 3. 모델 특성 비교

| 특성 | ALS | SASRec |
|------|-----|--------|
| 학습 방식 | 사용자-아이템 행렬 분해 | 시퀀스 패턴 학습 |
| 필요 데이터 | 사용자-아이템 상호작용 | **긴 시퀀스** |
| 희소 데이터 처리 | ✅ 내재적으로 우수 | ❌ 패턴 학습 어려움 |
| 이 데이터에 적합도 | **높음** | 낮음 |

## 💡 결론

> [!IMPORTANT]
> **이 데이터셋에서는 ALS가 SASRec보다 적합합니다.**

### 이유 요약

1. **짧은 시퀀스**: 47.5%가 5개 이하 인터랙션
2. **희소한 구매**: 전체의 0.02%만 구매
3. **view 중심**: 99.78%가 조회 이벤트 → 협업 필터링이 유리

### 권장 전략

1. ✅ **ALS 결과 사용** (현재 최고: 0.0955)
2. 🔧 **ALS 하이퍼파라미터 튜닝**으로 추가 개선 시도
3. 💡 필요시 ALS + 인기도 기반 앙상블 고려

## 📚 참고: SASRec이 효과적인 경우

SASRec은 다음 조건에서 효과적입니다:

- ✅ 긴 시퀀스 (20개 이상 인터랙션)
- ✅ 명확한 시간적 패턴 (예: 넷플릭스 시청 기록)
- ✅ 충분한 구매/전환 데이터

현재 데이터셋은 이 조건을 충족하지 않습니다.
