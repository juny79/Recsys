## 2. 주요 실험 분석

### 2.1 앙상블 성공 요인 (0.1196)
- **상호보완성:** ALS는 정적인 취향(CF)을, SASRec은 최근의 흐름(Sequential)을, XGBoost는 대중적 인기도(Popularity)를 각각 담당했습니다.
- **Soft Consensus:** 단순 가중합(Weighted Sum) 방식이 특정 모델의 오답에 휘둘리지 않고 '다수가 추천하는 아이템'을 상위로 올리는 데 효과적이었습니다.

### 2.2 LTR 모델의 정체 (0.1125)
CatBoost LTR이 앙상블 점수를 넘지 못한 근본적인 이유는 다음과 같습니다.
1.  **레이블의 편향 (Label Bias):** 학습 시 '마지막 인터랙션'만을 정답으로 사용했습니다. 하지만 실제 리더보드는 그 이후의 다양한 행동을 포함할 수 있어, 모델이 너무 '직전 행동'에만 과적합(Overfitting)되었을 가능성이 큽니다.
2.  **데이터 희소성:** 유저의 42%가 활동량 5건 미만인 상황에서, 고차원적인 '궤적(Trajectory)' 피처가 대다수 유저에게는 0(Zero) 값으로 채워져 모델 학습에 노이즈로 작용했습니다.
3.  **Heuristic의 강력함:** ALS의 `decay_rate=0.1`은 시간이 지남에 따라 지수적으로 가중치를 줄이는 매우 강력한 휴리스틱입니다. 머신러닝 모델이 피처만으로 이 복잡한 시간 함수를 완벽히 모사하기에는 데이터 양이 부족했습니다.

---

## 3. 결론 (Conclusion)

본 프로젝트를 통해 다음과 같은 실전적 통찰을 얻었습니다.
- **첫째,** 데이터가 희소한 커머스 환경에서는 **강력한 휴리스틱(Time Decay, Weighted Sum)**이 정교한 머신러닝 모델보다 때로는 더 견고한 성능을 보여줍니다.
- **둘둘째,** 모델의 복잡도를 높이기보다 **데이터의 질(Event Weighting)**과 **시간성(Recency)**을 튜닝하는 것이 점수 향상의 지름길이었습니다.
- **셋째,** CatBoost LTR은 라이트 유저에게는 노이즈를, 헤비 유저에게는 정확도를 제공하는 '양날의 검'이었습니다. 향후에는 유저별로 앙상블과 LTR을 섞어 쓰는 **Meta-Model** 방식이 유효할 것입니다.
